Services

The responsibilities of an angular service are as follows:
  • Handle non-view logic.
  • To communicate with the server.
  • To hold data and state.
  • In rare situations, you may do DOM manipulation.

Single Responsibility Principle

The SRP tells un that an object, in this case, should have a single responsibility.

Service Creation

There are several ways to create a service:

Value Function

valueStringSvc service output:{{valueStringSvc}}

valueObjSvc service output:{{valueObjSvc}}

Constant Function

The constant function is the same as the value function, but it's available at the application configuration stage.

The Factory Function

The factory function has to receive a function as it's second parameter and it executes this function and then caches the result, and whatever is the result of the function is returned every time someone asks for this service.

factoryObjSvc service output:{{factoryObjSvc}}

Service Function

The service function is similar to the factory function with one difference. The service function will accept a function as the second parameter except and will call new on it. It will then cache the result of that function and give you the result every time you call that service.

serviceObjSvc service output:{{serviceObjSvc}}

Provider Function

The provider function is what all the other service related functions use under the covers. It allow you to create a service at a much lower level.

A provider service must provide a $get function.

The benefit of creating services this way is that it allows the service to be configurable and allows you to configure it before it's created.

providerSvc service output:{{providerSvc}}

For more information on services, see the Writing Controllers & Services module from the Angular Best Practice course.
