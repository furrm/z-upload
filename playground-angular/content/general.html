General

Controllers

In AngulaJS, a controller is responsible for

  • Setting up the scope so data can be bound to the view.
  • Receiving action from the view, either internally, or marshalling an action off to the right place.
  • Should never be used to manipulate the DOM.

Services

Services do everything the other objects don't do. The term services is an overloaded term in IT, so below are a few examples of what services in AngularJS are used for:

  • Handle non-view related logic.
  • Contain business logic.
  • Used to communicate with the server.
  • Used to hold data and state, but this is not exclusive to services alone.
  • On very rare cases, services are used to manipulate the DOM.

Directives

Directives are by far the most complex object that AngularJS has to offer. This complexity is based on the fact that directives can contain a controller and a view. Below are the key responsibilities for directives:

  • DOM manipulation.
  • Receives interaction from the views (DOM).

Views

Views have 2 primary responsibilities:

  • Display the Application
  • Declare bindings and directives

Minification

Minification is used to:

  • Remove whitespace.
  • Shorten variable and parameter names.

Note:If you intend to minify your AngularJS script, you need to ensure that you make your script safe from the minification process.

Example 1: Unsafe script:

angular.module('app', []).controller('AppCtrl', function($scope){$scope.name = 'AppCtrl'});

Example 2: Safe script:

angular.module('app', []).controller('AppCtrl', ['$scope', function(scope){scope.name = 'AppCtrl'}]);

ngmin

ngmin is a utility that will go through any unsafe AngularJS script and make is safe for minification. ngmin ia a node utility, so to use it you will need node installed.

Use NPM to install ngmin.

$ npm install ngmin -g

The -g switch tells node to install the package globally.

To use ngmin, you need to tell specify a name for the input and output script files like so:

$ ngmin app.js app.safe.js

ngmin will prefer you to use a coding style where inline functions are used instead of external functions. To explain the better, see the following examples:

function MyController(){$scope.val = 'ngmin won\'t work';}

angular.module('app', []).controller('MyCtrl', MyController);


angular.module('app', []).controller('MyCtrl', function($scope){$scope.val = 'ng min will work =)'});
